git add(提交到共暂存区)

git commit (提交到版本区)

注意提交后需要写注释，在一个VIM文本编辑器页面；写完 注释；按esc,再输入：,再输入'wq';最后回到原来的命令页面

'git commit -a -m "注释"'(m 后面跟的是注释)（其实这个是直接从工作区提交到版本区等于提交的一个合写）

git log 查看提交的版本

git diff ''对比工作区与暂存区

git diff --cached '对比暂存区与工作区'

git diff master （master只是一个例子， 这个位置写一个分支或者仓库名）'对比工作区与版本区'

  • 撤销
  • git reset HEAD file name
  • git checkout -- file name
  • git commit -- amend
  • 有两个文件one.html two.html，被修改了，提交的时候 只提交了一个，忘了一个，那么我们该如何操作呢
  • git add one.html //先添加到暂存区 git commit -m "change1"//提交到工作区 git add two.html //先添加到暂存区 git commit -m "change1 one.html and two.html" -- amend //先把第一次提交的撤回到工作区再一起提交到工作区

删除

git rm file name(匹配工作区刚刚删除的文件，在暂存区也删除)

git rm -f file name(同时删除工作区和暂存区)

git rm --cached file name(只删除暂存区)

git branch git branch new1 git checkout new1 git checkout -b new2

合并两个分支 git status 这个是显示目前在哪个分支：On branch new1 git commit -a -m "change demo1.html" git checkout master （切换回master）(其实master就是一个指针) git merge new1
